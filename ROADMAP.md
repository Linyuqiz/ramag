# Ramag Roadmap

> 对标 DataGrip MySQL 的功能差距清单 + 实施路线图

最后更新：2026-04-26

## 目标定位

| 版本 | 目标 | DataGrip 覆盖率 | 周期 |
|------|------|---------------|------|
| **v0.1** | 个人日常能用，能取代 mycli + Sequel Ace 大部分场景 | ~15% | 4-6 个月 |
| **v0.2** | 加入 SSH/导出等专业开发者高频需求 | ~30% | +3 个月 |
| **v0.3** | 加入 PostgreSQL + 视图/函数浏览 | ~50% | +6 个月 |
| **未来** | 行业级开源工具，不做完全对标 DataGrip | — | — |

**重要**：明确放弃完全对标 DataGrip。一个人 5 年也做不到，不要内耗。

## 优先级标记

| 标记 | 含义 |
|------|------|
| 🔴 **必须**（Must-Have）| v0.1 没有这个就不能用 |
| 🟡 **重要**（Should-Have）| v0.2 加入，没有也能凑合 |
| 🟢 **优化**（Could-Have）| v0.3+ 加入，体验提升 |
| ⚪ **超纲**（Won't-Have）| 一个人做不到或 ROI 太低，明确不做 |

## 第五部分：明确不做（v1 内）

| ID | 功能 | 不做的理由 |
|----|------|-----------|
| W01 | 存储过程调试器 | 单步执行实现复杂度极高，使用频率极低 |
| W02 | AI 助手（自然语言→SQL） | 涉及外部 API 集成 + token 计费，且效果不如 GitHub Copilot 直接装 |
| W03 | Schema 对比 + 迁移脚本生成 | DataGrip 烧了多年，一个人做不出来好用版 |
| W04 | 数据脱敏 | 企业向，个人用户用不到 |
| W05 | 用户/权限管理 GUI | DBA 向，使用频率低，命令行更快 |
| W06 | MongoDB / Redis / 其他 NoSQL | 分别需要独立架构（zedis/pgpad 都只做一种）|
| W07 | 集群健康监控 | 这是 Grafana/Datadog 的领域 |
| W08 | 视图 + 触发器 + 函数 + 事件 编辑器 | 用户量小，DDL 文本编辑足够 |
| W09 | VCS（git）集成 | 用户用外部 IDE/git 客户端足够 |
| W10 | 团队协作 / 云同步设置 | 需要后端服务器，超出个人项目范围 |
| W11 | 重命名重构（改字段名传播） | 实现需要完整 SQL parser，工作量极大 |
| W12 | 全量 SQL 方言支持（Oracle/MSSQL） | 每种方言独立处理，扩散过广 |
